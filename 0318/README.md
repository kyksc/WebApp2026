# 📅 3월 18일 수업 실습: 리액트 빌드 및 호스팅

## 1. 과제 목표
- 리액트 프로젝트를 생성하고 가상 DOM의 원리를 이해함.
- 프로젝트를 빌드(Build)하고 GitHub Pages를 통해 실제 웹에 배포함.

## 2. 배포 URL
- [https://kyksc.github.io/WebApp2026/](https://kyksc.github.io/WebApp2026/)

## 3. 주요 실습 명령어
- `npm run build`: 프로덕션용 빌드 파일 생성
- `npm run deploy`: GitHub Pages에 정적 파일 호스팅

## 4. 학습 내용
- 브라우저 렌더링 과정(Parsing -> Style -> Layout -> Paint -> Composite) 이해

# 🚀 3주차 웹 프로그래밍 응용: 웹 동작 원리 및 React 심화

1.
엘리멘트 구조
1-1.
엘리멘트 구조의 이해
엘리멘트는 자바스크립트 함수로 만들어짐
엘리멘트는 돔트리로 구성되며, 각 노드는 태그와 1대1 매칭을 가짐
태그는 브라우저에 의해 돔이라는 자료형으로 만들어짐
태그 안에 다른 태그가 들어갈 수 있고, 중첩 가능함
노드 밑에 자식 노드가 있을 수 있음
1-2.
트리 구조와 렌더링
랜더링을 위해 HTML 태그와 CSS, 그리고 자바스크립트가 필요함
랜더링된 결과는 브라우저의 디스플레이 화면에 표시됨
렌더링을 위해서는 랜더트리 구조와 렌더링이 필요함
렌더링을 위해 자바스크립트가 필요하며, 랜더트리로 자바스크립트가 렌더링됨
모든 브라우저가 HTML 문서를 받아와서 렌더링 함
1-3.
리액트 구현
MPX 크리에이트 리액트 앱을 통해 가상 서버를 구축함
각 폴더의 의미를 이해해야 하며, '인덱스', '히트 이슈', '리액트'로 구분함
인덱스는 기본 파일로, 퍼블릭 디렉토리에 위치함
리액트 구현을 위해 프로젝트 이름을 지정하고, '에프리에트'로 설정함
'에프리에트'로 설정한 후 '코드'를 변경하면 즉시 렌더링됨
2.
자바스크립트와 리액트의 학습
2-1.
자바스크립트의 이해
자바스크립트는 인터랙션을 제어하는 툴임
워낙 체계적인 방식으로 이루어져 있어 이해가 간단하고 빠름
자바스크립트의 주요 요소는 시작 태그와 태그이며, 이들을 관통하는 것이 요소임
요소와 엘리멘트를 번역하는 게 요소라고 보며, 이는 엘리멘트의 개념과 비슷함
자바스크립트를 이용해 실제 화면에 그려지는 것은 돔에 의해 그려짐
2-2.
리액트의 이해와 엘리멘트 생성
리액트에서의 엘리멘트는 가상돔에 있는 노드를 의미함
엘리멘트 생성 시, 자바스크립트의 요소 생성과 유사하게 적용되나, 그 과정이 다름
리액트에서는 리액트 엘리멘트 생성 함수를 이용해 엘리멘트 생성을 간편하게 함
마찬가지로, 리액트에서도 엘리멘트는 가상돔에 존재하고, 버추얼 돔이나 현실 돔에서 형태가 유사하게 나타남
버추얼 돔에서 존재하는 엘리멘트를 리액트 엘리멘트라고 함
2-3.
컴포넌트의 이해와 활용
컴포넌트는 화면에 표현되는 하나하나를 의미함
커미 컴포넌트와 펑션 컴포넌트의 두 가지 종류가 있음
펑션 컴포넌트는 실제 엘리멘트를 생성하는 함수를 의미함
복잡한 로직을 단순화하고 코드를 간편하게 만드는데 사용됨
따라서 펑션 컴포넌트는 리액트에서 중요한 역할을 하며, 실제 엘리멘트를 만드는 핵심 요소임
3.
UI 컴포넌트
3-1.
UI 컴포넌트 이해
UI 블록의 작은 단위인 엘리멘트에 대해 설명함
엘리멘트 생성 후 변경 불가능한 속성과, 생성자 및 수정자에 대해 설명함
변경 시 카운캐스트 앤 다이렉트 앤 패브리케이션을 통해 전체 페이지가 아닌 부분 변경을 확인함
변화 발생 시 실제 페이지 변경이 아닌 부분별로 전부 변경을 확인함
리액트 엘리멘트만의 특징인 변화 여부와 메모리 내 가상 도움 생성 과정을 설명함
3-2.
컴포넌트 생성과 수정
컴포넌트 내 엘리멘트 생성과 수정 과정에 대해 설명함
컴포넌트 생성 시 속성 변경과 관련된 컨텍스트 앤 바이어스를 소개함
리액트 엘리멘트만의 기능인 카운캐스트 앤 다이렉트 앤 패브리케이션을 소개함
화면 변경 시 카운캐스트 앤 다이렉트 앤 패브리케이션을 통해 실제 페이지 변경을 확인함
기존 엘리멘트와 새 엘리멘트의 변경 여부를 비교하고, 변경 시 카운캐스트 앤 다이렉트 앤 패브리케이션을 통해 확인함
3-3.
렌더링과 업데이트
화면에 렌더링된 요소 변경 시 카운캐스트 앤 다이렉트 앤 패브리케이션을 통해 확인함
사용자 클릭 이벤트 발생 시 페이지 변경 여부를 확인하고, 이벤트 발생 시 렌더링된 가상 도움을 생성함
화면 일부 변경 시, 해당 부분만 렌더링되어 새로운 가상 도움을 생성함
새로운 가상 도움과 기존 가상 도움을 비교하여 변경된 부분을 찾고, 해당 부분만 업데이트함
이러한 과정을 카운캐스트 앤 다이렉트 앤 패브리케이션으로 표현함
4.
웹 개발 기초 이해 - 웹 애플리케이션과 리액터
4-1.
웹 개발과 리액터의 기본 이해
웹 개발에서 가장 중요한 도구인 리액터에 대해 설명함
리액터는 가상의 도움을 메모리로 잡아놓고 있어 단점이 있음
하지만 리액터를 사용하면 화면이 빠르게 바뀌는 속도를 보여줌
인덱션에 사용되는 DIV(직접 부여 아이디)에 대해 설명함
리액터의 시작점인 ID는 루트와 함께 웹 애플리케이션의 리액터로 사용됨
4-2.
블록과 인라인 요소 이해
블록과 인라인은 웹 개발에서 사용하는 두 가지 주요 요소임
블록은 하나의 컨텐츠를 표현하며, 인라인은 여러 개의 컨텐츠를 표현함
블록은 라인별로 컴텐츠를 표현하며, 인라인은 콘텐츠를 분리하여 표현함
이러한 이해를 통해 웹 개발에서의 '블록'과 '인라인'의 차이를 명확히 파악할 수 있음
4-3.
리액터 애플리케이션 생성
리액터로 웹 페이지의 콘텐츠를 생성할 때, 필요한 요소들을 미리 정의하고, 이를 이용하여 페이지를 구성함
이때 사용하는 기능들에 대해 설명하고, 각각의 역할에 대해 이해함
이 과정을 통해 실제 웹 페이지의 구조와 동작 원리를 이해하게 됨
이를 바탕으로 실제 웹 페이지의 생성과 수정을 할 수 있게 됨
5.
리액트 활용
5-1.
리액트의 기본 구조
리액트는 화면에 표현하고 싶은 대로 구성 가능함
리액트는 탭을 만들고, 리액트 태그에 컨텍스트를 지정하여 화면에 표시함
리액트는 엘리멘트 생성 함수로 이루어져 있음
리액트 태그를 감싸는 엘리멘트를 클락 컴포넌트라고 부름
클락 컴포넌트는 리액트 태그를 만드는 함수임
5-2.
클락 컴포넌트의 활용
클락 컴포넌트는 리액트 태그를 만들고, 이를 원하는 대로 화면에 표시함
클락 컴포넌트는 클락이나 컴포넌트 태그를 만들어 원하는 요소를 표시할 수 있음
클락 컴포넌트는 화면에 표시된 대로 클릭하면 기능을 수행함
클락 컴포넌트를 사용하면 코드 중 클락 컴포넌트 관련 내용을 지우지 않아도 됨
클락 컴포넌트를 지우고 싶으면 클락 컴포넌트를 지정해야 함
5-3.
리액트 코드 작성
클락 컴포넌트를 만드는 코드를 리액트 코드라고 부름
리액트 코드는 태그 안에 리액트 태그를 만드는 엘리멘트로 구성됨
클락 컴포넌트를 지정한 후, 지정된 엘리멘트에 리액트 태그를 삽입함
이 과정을 통해 원하는 요소를 화면에 표시할 수 있음
클락 컴포넌트를 이용한 리액트 코드 작성은 프론트 앤드의 특징 중 하나임
6.
리액트와 자바스크립트
6-1.
리액트와 자바스크립트 소개
리액트는 웹 개발에 사용되는 빌드 인 가이드, 웹 페이지를 자세히 보기 위해
자바스크립트는 프로그래밍 언어로서, 웹 페이지의 동적 성능을 개선
리액트와 자바스크립트의 장점은 웹 페이지의 빠른 실행과 유저 경험 향상
리액트는 파이썬과 유사한 프로그래밍 언어로, 파이썬은 C 언어와 유사
리액트는 페이지 별로 코드를 관리하기 위해, 페이지 마다 관리하는 객체를 생성
6-2.
백그라운드 이해
백그라운드는 리액트가 웹 페이지를 생성하는 데 필요한 환경을 의미
리액트는 웹 페이지의 복잡한 구조를 단순화하므로, 웹 개발에 용이
백그라운드는 웹 페이지가 어떻게 렌들되는지, 어떤 내용을 포함하는지 등을 결정
백그라운드는 실제 웹 페이지를 생성하거나, 웹 페이지를 생성하는 데 필요한 프로그래밍 언어를 포함
백그라운드가 웹 페이지 생성에 필요한 컴파일러 정보를 제공하므로, 웹 페이지 생성 시 빠르게 실행 가능
6-3.
개발 테스트와 결과
개발 테스트는 리액트가 웹 페이지를 생성하고, 테스트가 이 웹 페이지가 올바르게 생성되었는지 확인하는 과정
테스트는 웹 페이지 생성 시, 웹 페이지의 동작 여부를 검증하는 과정
테스트는 메소드 테스트와 스페이스 테스트로 나뉘며, 메소드 테스트는 페이지의 일부만 테스트, 스페이스 테스트는 전체 페이지 테스트
테스트 결과는 리액트의 성능을 평가하고, 개발자에게 피드백을 제공
테스트 결과가 예상과 다르면, 개발자는 코드를 수정하여 테스트 결과를 개선해야 함
7.
자바스크립트 이해
7-1.
렌더링과 자바스크립트 기초
가급적 피하는 게 좋음, 힘들어도 한번 더 생각해 보라고 권장함
타인의 입장 생각해보고, 싫어하는 사람에게 표정과 행동이 어떻게 다른지 관찰해보라고 함
상대방의 입장을 생각해보고, 자세히 얘기하며, 정확하게 얘기하는 게 중요하다고 강조함
하지만 아이들에게는 무조건 맞춰주기보다는 진도를 나가면서 발표할 수 있게 해야 한다고 조언함
렌더링을 통해 화면 전체가 아닌 특정 부분만 처리하는 방법을 설명함
7-2.
메모리 모델과 함수
메모리 모델과 함수, 클래스, 메뉴형 스타일을 소개함
메모리 모델에는 메모리 모델 클래스와 메모리 모델 함수가 있음
자바스크립트 함수의 중요성과 객체의 개념을 강조함
객체를 만들 때 선형형과 메모리형에 대해 설명함
강의에서는 메모리 모델과 함수에 대한 발표를 강조함
7-3.
렌더링과 디렉토리
렌더링과 디렉토리에 대해 설명함
렌더링을 통해 간단한 자기소개 웹페이지를 만들 수 있다고 예시를 들어 설명함
렌더링과 러버블을 이용한 자기소개 웹페이지 만들기를 강조함
러버블을 이용한 포트폴리오 만들기를 추천함
발표자 외 다른 사람의 발표를 독려하며, 일주일 안에 러버블을 배포할 수 있다면 충분하다고 조언함
8.
소스트리 설치 및 연동
8-1.
소스트리 소개
소스트리는 코드 오류 시 쉽게 취소하고 과거 원하는 시점으로 돌아갈 수 있음
과거의 코드와 현재 코드 비교 가능, 브랜치 기능으로 독립적인 공간 생성 가능
브랜치는 수정한 내용은 본 작업물에 병합되며, 병합 확인 후 커밋으로 저장
소스트리는 변환 메모를 통해 어떤 변경 사항이 발생했는지 기록 가능
소스트리는 직관적이고 편리하게 병합을 시각화해줌
8-2.
소스트리 설치 방법

9.
웹 개발 툴 설명
9-1.
테스트 및 연동 옵션
'연동된 테스트' 메뉴에 들어가면 '폴더 연동'을 통해 다른 서버에 테스트를 올림
'연동된 서버'에서 '연동' 버튼을 눌러서 테스트용 폴더를 생성함
테스트용 폴더 안의 이미지 파일을 선택하여 '연동' 버튼을 눌러서 테스트를 시작함
'연동' 버튼을 눌렀을 때, 프라이빗 테스트와 대위 테스트가 생성됨
테스트용 폴더에 이미지 파일을 넣으면, 해당 파일이 테스트용 폴더로 생성되고 다른 이미지 파일을 넣을 수 있음
9-2.
메시지 전달 및 변경 기능
커밋 메시지를 입력할 수 있는란 입력할 수 있는 곳에 '체인지스' 대문자를 넣어 메시지 입력 가능
커밋에서 첫 글자만 대문자로 돼 있는 '체인지스'를 클릭해서 메시지를 입력함
테스트용 폴더에 이미지 파일을 넣어 메시지 입력을 변경할 수 있음
테스트용 폴더에 변경된 메시지를 넣으면, 다른 이미지 파일도 변경됨
'연동' 버튼을 눌렀을 때, 변경 기능이 작동하여 메시지 입력할 수 있음

웹 개발 툴
'웨이브'는 웹 개발 툴 중 하나로, 과거의 아파치 웹 서버를 대체함
웹 개발 환경에서 쓰이는 라이브러리, 클래스, 메소드를 포함한 웹 개발 프레임워크임
'웨이브'는 '노드'와 'jin'을 포함한 여러 웹 개발 툴을 포괄적으로 관리함
웹 개발 환경에서 쓰이는 라이브러리, 메소드 등을 포함한 웹 개발 프레임워크임
웹 개발 툴 중 하나로, '노드'와 'jin'을 포함한 웹 개발 툴을 포괄적으로 관리함
