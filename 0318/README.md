# 🚀 3주차 웹 프로그래밍 응용: 웹 동작 원리 및 React 심화

## 1. 웹 동작 및 렌더링 기초
* **클라이언트와 서버:** 사용자의 UI를 담당하며 요청(Request)을 보내는 클라이언트와, 데이터를 처리하고 응답(Response)을 반환하는 서버 구조로 동작합니다.
* **브라우저 렌더링 5단계:**
  1. **파싱(Parsing):** HTML을 파싱해 DOM 트리를, CSS를 파싱해 CSSOM 트리를 생성합니다.
  2. **스타일(Style):** DOM과 CSSOM 트리를 결합하여 렌더(Render) 트리를 생성합니다.
  3. **레이아웃(Layout):** 요소들의 정확한 위치와 크기를 계산합니다.
  4. **페인트(Paint):** 실제 픽셀로 변환하여 그립니다.
  5. **합성(Compositing/Display):** 화면에 최종적으로 출력합니다.
* **자바스크립트(JS) 위치:** `<script>` 태그는 브라우저의 DOM 렌더링을 방해하지 않도록 `<body>` 태그의 가장 하단에 두는 것이 좋습니다.
* **ECMAScript:** 자바스크립트의 표준 명칭으로, 현업에서는 동일한 개념으로 사용됩니다.

---

## 2. 웹 애플리케이션 및 서버 개념
* **SPA vs MPA:**
  * **SPA** (Single Page Application): 페이지를 하나만 두고, 자바스크립트를 이용해 동적으로 필요한 부분만 변경합니다. 속도가 빠르고 사용자 경험이 좋습니다.
  * **MPA** (Multi Page Application): 페이지를 이동할 때마다 전체 HTML을 서버에서 새로 받아오는 전통적인 방식입니다.
* **웹 서버:** 클라이언트 요청을 처리하는 소프트웨어로 IIS, Tomcat, Apache 등이 있으며 최근에는 Node.js 기반 환경도 널리 쓰입니다.
* **소프트웨어 배포 관련 용어:**
  * **Release:** 새로운 버전을 배포하거나 새 제품을 만드는 것.
  * **Deploy:** 프로그램을 서버에 설치하여 서비스 제공 준비를 하는 것.
  * **Distribute:** 제품을 사용자들이 실제 사용할 수 있도록 서비스를 제공하는 것.

## 3. 리액트(React) 핵심 정리
* **개요:** 사용자 인터페이스(UI)를 만들기 위해 만들어진 컴포넌트 기반의 JavaScript 라이브러리입니다.
* **Virtual DOM (가상 돔):**
  * 메모리에 존재하는 가상의 DOM입니다.
  * 상태 변화가 발생하면 이전 가상 돔과 새로운 가상 돔을 비교(Diffing)하여 변경된 부분만 실제 DOM에 업데이트합니다. 화면 전체를 다시 그리지 않아 렌더링 속도가 매우 빠릅니다.
* **JSX (JavaScript XML):**
  * 자바스크립트 내에서 HTML 문법을 사용할 수 있게 해주는 확장 문법입니다.
  * 코드 가독성이 좋으며, 바벨(Babel) 컴파일러에 의해 `React.createElement()` 자바스크립트 코드로 변환됩니다.
* **Element vs Component:**
  * **Element (엘리먼트):** 화면에 표시될 내용을 담고 있는 리액트의 가장 작은 기본 단위이며 불변(Immutable) 객체입니다.
  * **Component (컴포넌트):** 여러 엘리먼트를 모아 독립적이고 재사용이 가능하게 만든 UI 모듈(함수형 또는 클래스형)입니다.
