## 1. 🌐 웹 기초 및 렌더링 프로세스

### 📌 메타데이터 & SEO
* **메타데이터:** 데이터를 설명하기 위한 데이터로, 검색엔진 로봇이 사이트를 수집할 때 가중치를 결정하는 기준이 됩니다.
* **SEO (검색엔진 최적화):** 메타태그를 적절히 활용하여 검색엔진 노출 순위를 높이는 기술입니다.

### 🌳 DOM (Document Object Model)
* HTML 문서를 트리 형태의 객체로 표현한 모델입니다.
* JavaScript를 통해 DOM 객체(노드/엘리먼트)에 접근하여 스타일, 텍스트 등을 동적으로 제어합니다.

### 🎨 렌더링 파이프라인 (Critical Rendering Path)
글자로 이루어진 HTML이 브라우저 화면에 시각적으로 그려지는 전체 과정입니다.
1. **Parsing:** HTML과 CSS를 읽어 각각 `DOM 트리`와 `CSSOM 트리`를 생성
2. **Render Tree:** DOM과 CSSOM을 결합
3. **Layout:** 각 노드의 정확한 위치와 크기를 계산 (반응형 웹의 핵심)
4. **Paint:** 계산된 요소를 실제 픽셀로 변환
5. **Composite:** 여러 레이어를 합성하여 최종 화면 출력
> 💡 **최적화 팁:** DOM 변경 시 레이아웃을 다시 계산하는 **Reflow**와 다시 그리는 **Repaint**를 최소화해야 성능이 향상됩니다. 또한 브라우저 렌더링 차단을 막기 위해 `<script>` 태그는 `<body>` 최하단에 배치하는 것을 권장합니다.

## 2. ⚛️ React & Virtual DOM

### 💡 React 핵심 특징
Facebook(Meta)에서 개발한 **사용자 인터페이스(UI) 구축용 JavaScript 라이브러리**입니다.
* **컴포넌트 기반:** 블록 조립하듯 개발하여 재사용성과 유지보수성이 뛰어납니다.
* **SPA (Single Page Application):** 단일 페이지 안에서 필요한 콘텐츠만 부분적으로 업데이트하여 쾌적한 UX를 제공합니다. (전체 새로고침이 일어나는 MPA 방식과 대비됨)

### 🚀 Virtual DOM (가상 돔)
* **동작 원리:** 메모리상에 가상의 DOM을 복제해 둡니다. 상태(State) 변경 시 새로운 가상 DOM을 만들고, 이전 가상 DOM과 비교(**Diffing**)합니다.
* **장점:** 전체 화면을 다시 그리지 않고 **변경된 부분만 실제 DOM에 한 번에 적용**하므로 렌더링 속도가 압도적으로 빠릅니다.

### 📝 JSX (JavaScript XML)
JavaScript 코드 내부에 HTML 문법을 직관적으로 작성할 수 있게 해주는 확장 문법입니다.
* **동작:** 브라우저가 읽기 전 **Babel** 컴파일러를 통해 `React.createElement()` 순수 JS 코드로 변환됩니다.
* **필수 규칙:** 1. 무조건 **하나의 부모 요소**(`<></>` 빈 태그 등)로 전체를 감싸야 합니다.
  2. JavaScript 표현식 및 변수는 중괄호 `{}` 안에 작성합니다.

## 3. 🛠 개발 생태계 & 인프라

### 🖥 서버 vs 클라이언트
* **클라이언트:** 브라우저 등에서 요청(Request)을 보내는 주체 (HTML, CSS, JS 활용)
* **서버:** 요청을 받아 데이터/웹페이지를 응답(Response)하는 주체 (Node.js, Tomcat, Apache 등)
* **포트(Port) 번호:** 하나의 컴퓨터 내에서 여러 서버를 구분하는 식별 번호 (예: `localhost:3000`)

### 📦 빌드 및 패키지 관리 도구
* **npm:** Node.js 기반 패키지 설치 및 관리 도구 (`npm start`: 개발 서버 실행, `npm run build`: 배포 파일 생성)
* **npx:** 패키지를 임시로 다운로드해 일회성으로 실행하는 도구 (`npx create-react-app`)
* **Webpack:** 개발 중인 수많은 파일을 배포하기 좋은 형태로 하나로 묶어(Bundle) 최적화해 주는 도구입니다.

### ☁️ 언어 & 프레임워크 추가 지식
* **TypeScript (`.ts`, `.tsx`):** 자유로운 JS에 '정적 타입(Type)'을 강제하여 에러를 방지하는 컴파일러형 언어입니다.
* **SCSS/SASS:** 기존 CSS에 변수, 중첩, 연산 등 프로그래밍 기법을 추가해 효율을 높인 스타일 시트입니다.
* **호스팅(Hosting):** 자체 서버 없이 AWS, Github Pages 등의 클라우드 트래픽/하드웨어를 빌려 서비스를 배포(Deploy)하는 방식입니다.

## 4. 📁 React 프로젝트 구조

`npx create-react-app [프로젝트명]` 실행 시 생성되는 핵심 구조입니다.

* `node_modules/` : npm으로 설치된 외부 패키지 모음
* `public/index.html` : 뼈대가 되는 단일 HTML 템플릿 파일
* `src/index.js` : 메인 프로그램의 시작점 (App 컴포넌트를 HTML에 연결)
* `src/App.js` : **실제 화면에 표시될 메인 UI 컴포넌트 (주요 작업 공간)**
* `.gitignore` : 용량이 큰 `node_modules` 등을 Git 업로드 목록에서 제외하는 설정 파일
