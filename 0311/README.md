# 🚀 2주차 웹 프로그래밍 응용: React 기초 및 개발 환경

## 1. 웹 개발 기초 및 아키텍처
* **HTML과 메타데이터 (Meta Data):** HTML은 웹페이지의 뼈대를 구성하며, 메타데이터는 이 데이터(웹페이지)를 설명하는 숨은 정보입니다.
* **SEO (검색엔진 최적화):** `<head>` 영역에 메타 태그를 적절히 배치하면 검색엔진이나 AI 봇의 노출 빈도를 높일 수 있습니다.
* **웹 표준 기관:** W3C, MDN, WHATWG 등에서 웹에 대한 표준 규격을 확인하고 공부할 수 있습니다.

### 🔄 MPA vs SPA
| 구분 | 특징 | 예시 |
| :--- | :--- | :--- |
| **MPA** (Multiple Page Application) | 클릭 시 완전히 새로운 HTML 페이지를 불러와 전체 화면이 바뀌는 전통적인 방식입니다. | 대학 홈페이지 등 |
| **SPA** (Single Page Application) | 단일 HTML 파일 안에서 필요한 부분(컴포넌트)만 동적으로 바꿔주는 방식입니다. 리액트(React)가 주로 사용합니다. | 최신 웹 애플리케이션 |

## 2. 개발 환경과 Node.js
* **Node.js:** 자바스크립트는 본래 브라우저 안에서만 동작했지만, Node.js를 통해 서버 측이나 로컬 컴퓨터 어디서든 자바스크립트를 실행할 수 있는 **런타임(Runtime)** 환경이 마련되었습니다.
* **IDE (통합 개발 환경):** VS Code는 기본적으로 '코드 편집기'이지만, 확장 플러그인을 설치함으로써 IDE(디버깅, 빌드 등 지원)의 역할을 수행합니다.

### 💡 환경 관련 핵심 용어
* **Platform:** 소프트웨어가 실행될 수 있는 기반 환경 (예: 운영체제)
* **Environment:** 소프트웨어 실행을 위해 구성된 조건
* **Framework:** 개발을 위한 뼈대 및 기반 구조 (예: .NET)
* **Runtime:** 프로그램이 실행되는 환경 (예: JVM, Node.js)

## 3. 패키지 매니저 & 빌드 도구

### 📦 npm vs npx
| 구분 | 역할 | 예시 명령어 |
| :--- | :--- | :--- |
| **npm** (Node Package Manager) | 프로젝트의 의존성(패키지)을 지속해서 **설치하고 관리**합니다. | `npm install axios` |
| **npx** (Node Package Execute) | 패키지를 굳이 설치하지 않고, 최신 버전으로 **일회성 실행**을 할 때 사용합니다. | `npx create-react-app my-app` |

### ⚡ Vite vs Webpack
| 구분 | 특징 |
| :--- | :--- |
| **Vite** (권장) | ES Module 기반으로 동작하여 번들링 속도가 매우 빠르고 설정이 간단해 최근 가장 권장되는 빌드 도구입니다. |
| **Webpack** | 전체 파일을 번들링하는 방식으로 초기 구동이 느리고 설정이 복잡합니다. |

## 4. 리액트(React) 핵심 및 프로젝트 관리
* **가상 돔(Virtual DOM)의 장점:** 기존 웹이 화면 전체의 DOM을 다시 그렸다면, 리액트는 메모리상의 '가상 돔'을 사용해 변경된 부분만 찾아 실제 화면에 반영하므로 렌더링 속도가 매우 빠릅니다.

### 🛠️ 프로젝트 생성 명령어
* **Vite 방식 (권장):** `npm create vite@latest my-app --template react`
* **CRA 방식:** `npx create-react-app my-app`

> **🚨 Git 업로드 시 주의사항 (`.gitignore`)**
> 리액트 프로젝트 생성 시 만들어지는 `node_modules` 폴더는 용량이 매우 큽니다. 따라서 깃허브에 업로드할 때는 `.gitignore` 파일에 이를 포함시켜 업로드 대상에서 제외해야 합니다.

## 5. JSX (JavaScript XML) 문법 핵심
* **개념:** JavaScript 코드 내부에서 HTML과 유사한 마크업 문법을 사용하여 직관적으로 UI를 짤 수 있게 해주는 확장 문법입니다.
* **Babel:** 브라우저는 JSX를 바로 읽을 수 없으므로, Babel이라는 도구가 이를 일반적인 JavaScript 코드(`React.createElement()`)로 변환해 줍니다.

### 📝 필수 문법 및 규칙
1. **반드시 단일 부모 요소로 감싸기:** 2개 이상의 태그를 반환(`return`)할 경우, 전체를 감싸는 묶음 태그(또는 `<>` `</>` 빈 태그)가 없으면 에러가 발생합니다.
2. **자바스크립트 표현식 사용:** 중괄호 `{}`를 사용하면 그 안에서 자바스크립트 변수나 수식을 사용할 수 있습니다.
3. **카멜 표기법(camelCase):** 기존 HTML의 `class` 대신 `className`을 사용해야 합니다.
4. **조건부 렌더링:** 삼항 연산자(`조건 ? A : B`)나 논리 연산자(`&&`)를 활용해 상황에 맞는 UI를 쉽게 그릴 수 있습니다.
