## 1. 엘리멘트(Element) 구조

### 1-1. 엘리멘트 구조의 이해
* **자바스크립트 함수 기반:** 엘리멘트는 자바스크립트 함수로 만들어집니다.
* **DOM 트리 매칭:** 엘리멘트는 DOM 트리로 구성되며, 각 노드는 HTML 태그와 1:1 매칭을 가집니다.
* **DOM 자료형 변환:** 태그는 브라우저에 의해 'DOM'이라는 자료형으로 만들어집니다.
* **계층 구조:** 태그 안에 다른 태그가 들어갈 수 있어 중첩이 가능하며, 노드 밑에 자식 노드를 가질 수 있습니다.

### 1-2. 트리 구조와 렌더링
* **렌더링 필수 요소:** 렌더링을 위해 HTML 태그, CSS, 자바스크립트가 필요합니다.
* **디스플레이 표시:** 렌더링된 결과는 브라우저의 화면에 표시됩니다.
* **렌더 트리(Render Tree):** 렌더링을 위해서는 렌더 트리 구조가 필요하며, 자바스크립트가 렌더 트리를 기반으로 렌더링을 수행합니다.

### 1-3. 리액트(React) 구현
* **가상 서버 구축:** `npx create-react-app`을 통해 가상 서버를 구축합니다.
* **디렉토리 구조 이해:** `index`, `public` 등 각 폴더의 의미를 이해해야 합니다. (`index`는 기본 파일로 `public` 디렉토리에 위치합니다.)
* **즉시 렌더링(Hot Reloading):** 프로젝트 생성 후 코드를 변경하면 화면에 즉시 렌더링되어 반영됩니다.

---

## 2. 자바스크립트와 리액트의 학습

### 2-1. 자바스크립트의 이해
* **인터랙션 제어:** 자바스크립트는 웹상의 인터랙션을 제어하는 도구입니다.
* **구조적 접근:** 체계적인 방식으로 이루어져 있어 이해가 빠릅니다.
* **요소(Element)의 역할:** 시작 태그와 종료 태그를 관통하는 것이 요소이며, 자바스크립트를 이용해 실제 화면에 그려지는 것은 DOM에 의해 처리됩니다.

### 2-2. 리액트의 이해와 엘리멘트 생성
* **가상 돔(Virtual DOM):** 리액트에서의 엘리멘트는 가상 돔에 있는 노드를 의미합니다. (이를 리액트 엘리멘트라 부릅니다.)
* **생성 과정의 차이:** 자바스크립트의 요소 생성과 유사하지만 그 과정이 다릅니다. 리액트 엘리멘트 생성 함수를 이용해 보다 간편하게 생성합니다.

### 2-3. 컴포넌트(Component)의 이해와 활용
* **컴포넌트의 정의:** 화면에 표현되는 독립적인 UI 조각을 의미합니다.
* **함수형 컴포넌트(Function Component):** 실제 엘리멘트를 생성하는 함수를 의미하며, 복잡한 로직을 단순화하고 코드를 간편하게 만듭니다. (리액트에서 엘리멘트를 만드는 핵심 역할)

---

## 3. UI 컴포넌트

### 3-1. UI 컴포넌트 이해
* **불변성:** 엘리멘트 생성 후에는 속성 변경이 불가능합니다.
* **부분 변경 (Reconciliation 과정):** 상태 변화 발생 시 실제 페이지 전체가 아닌, 가상 돔 메모리 내에서 변경된 부분만 파악하여 업데이트합니다.

### 3-2. 컴포넌트 생성과 수정
* **가상 돔 비교:** 사용자 클릭 등의 이벤트로 화면이 변경될 때, 새로운 가상 돔을 생성합니다.
* **변경점 파악:** 기존 가상 돔과 새 가상 돔을 비교하여 변경된 요소와 속성만 정확히 찾아냅니다.

### 3-3. 렌더링과 업데이트
* 이벤트 발생 시 부분 렌더링을 통해 새로운 가상 돔이 생성되고, 비교 과정을 거쳐 실제 필요한 부분만 최종 업데이트됩니다.

---

## 4. 웹 애플리케이션과 리액트 기초

### 4-1. 웹 개발과 리액트의 기본 이해
* **빠른 속도:** 리액트는 메모리에 가상 돔을 올려두고 비교하므로 화면 전환 및 업데이트 속도가 매우 빠릅니다.
* **시작점 (Root ID):** 웹 애플리케이션의 렌더링 시작점으로 특정 `div`(보통 `id="root"`)가 사용됩니다.

### 4-2. 블록과 인라인 요소 이해
* **블록(Block) 요소:** 하나의 콘텐츠가 전체 라인(영역)을 차지하며 표현됩니다.
* **인라인(Inline) 요소:** 여러 개의 콘텐츠가 한 라인 안에서 분리되어 나란히 표현됩니다.

### 4-3. 리액트 애플리케이션 생성
* 리액트로 웹 페이지를 구성할 때 필요한 요소들을 미리 정의하고, 기능들의 역할을 이해하여 실제 페이지의 구조와 동작 원리를 파악합니다.

---

## 5. 리액트 활용

### 5-1. 리액트의 기본 구조
* 원하는 컴포넌트를 만들고 컨텍스트를 지정하여 화면에 표시할 수 있습니다.
* 컴포넌트는 리액트 엘리멘트 생성 함수로 이루어져 있습니다.

### 5-2. 컴포넌트의 활용
* 원하는 요소를 독립적으로 생성하고 화면에 배치하여 각자의 기능을 수행하게 할 수 있습니다.
* 모듈화가 잘 되어 있어, 코드의 특정 부분을 지우거나 수정할 때 해당 컴포넌트만 제어하면 됩니다.

### 5-3. 리액트 코드 작성
* 컴포넌트를 선언하고 지정된 엘리멘트를 DOM에 삽입하는 과정은 최신 프론트엔드 개발의 핵심 특징입니다.

---

## 6. 리액트와 자바스크립트 환경

### 6-1. 백그라운드 이해
* **실행 환경:** 리액트가 웹 페이지를 생성하는 데 필요한 컴파일 및 렌더링 환경을 의미합니다.
* 복잡한 구조를 단순화하여 웹 개발을 용이하게 만들어줍니다.

### 6-2. 개발 테스트와 결과
* 생성된 웹 페이지가 올바르게 동작하는지 검증하는 과정입니다.
* 테스트 결과를 바탕으로 리액트의 성능을 평가하고, 예상과 다를 경우 코드를 수정하여 개선합니다.

---

## 7. 자바스크립트 심화 이해

### 7-1. 렌더링 기초 및 태도
* 타인의 입장(사용자 경험)을 생각하며 정확하고 직관적인 개발을 하는 것이 중요합니다.
* 화면 전체를 다시 그리는 것을 피하고 부분 렌더링을 처리하는 방법을 숙지해야 합니다.

### 7-2. 메모리 모델과 함수
* 자바스크립트의 함수와 객체(Object) 개념이 매우 중요합니다.
* 메모리 상에서 객체와 함수가 어떻게 할당되고 참조되는지 이해해야 합니다.

---

## 8. SourceTree(소스트리) 설치 및 연동

### 8-1. 소스트리 소개
* **버전 관리 시각화:** 코드 오류 시 과거로 쉽게 돌아가거나(Revert/Reset), 변경 사항을 직관적으로 확인할 수 있는 Git GUI 툴입니다.
* **브랜치(Branch):** 독립적인 작업 공간을 만들고, 작업 완료 후 본 작업물에 병합(Merge)할 수 있습니다.

### 8-2. 소스트리 사용 방법
* 변경된 파일을 스테이지에 올리고(Stage), 메모를 작성하여 커밋(Commit)합니다.
* 직관적인 UI를 통해 브랜치 생성, 커밋 내역 확인, 병합 상태 등을 쉽게 관리할 수 있습니다.

### 8-3. GitHub 연동
* 소프트웨어 개발 프로젝트 관리 및 협업을 위한 대표적인 플랫폼입니다.
* 버전 관리, 원격 저장소, 협업, GitHub Pages 기능 등을 제공합니다.

---

## 9. 웹 개발 툴 설명

### 9-1. 테스트 및 연동 옵션
* 로컬 폴더와 서버를 연동하여 결과물을 테스트 환경에 쉽게 배포하고 확인할 수 있습니다.

### 9-2. 커밋 메시지 작성 규약

### 9-3. 기타 웹 개발 프레임워크 툴
* 노드(Node.js)를 포함하여 패키지, 라이브러리, 서버 등을 포괄적으로 관리할 수 있는 다양한 개발 환경 툴을 활용합니다.
